Getting Started with HasDocs

  • Step 1: Sign up
  • Step 2: Write the docs
  • Step 3: Import a repository
  • Step 4: Deploy the docs

Step 1: Sign up

Sign up using your existing GitHub account. HasDocs requires permission to read from your public and private repos.

Step 2: Write the docs

Write the documentation for your project. HasDocs supports Sphinx, Jekyll, RDoc, and Javadoc.

Step 3: Import a repository

Sync your GitHub repositories by clicking the sync button on your account page. Then you can install a a post-receive hook to the repo by clicking the create hook button.

Step 4: Deploy the docs

Deploy the docs by pushing to the GitHub. Your docs are built by the appropriate documentation generator and hosted on HasDocs.
