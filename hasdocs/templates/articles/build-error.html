Common issues

Build can fail for many reasons. The following are the most common cases for build failure.

Documentation or requirements path

For Sphinx projects, docs is the most common path for documentation. For pip, optional requirements.txt is the common filename for dependencies. Many Jekyll projects have the current directory . as the docs path.

Modules with C extensions

It is also possible that your project has some modules with C extensions. Most libraries for web applications are available at build time.

If a shared library need for a module, however, isn't available, please contact us at support@hasdocs.com for help.

Absolute binary path

If you specify an absolute path (/User/foo/bin/sphinx-build, for example) for a binary file in Makefile or other scripts, build may fail after failing to locate the binary on the build system.

Seeing the build logs

You can see the output from build at www.hasdocs.com/owner/project/logs/.

Generator specific issues

Sphinx

For Sphinx docs, a virtualenv is created. All the dependencies are installed by running python setup.py or pip -r requirements.txt if a requirement file is specified.

Then the docs are built by running make html in the docs directory specified for the project.

We use Sphinx version 1.1.3.

Jekyll

Jekyll docs are built by running jekyll --pygments --no-lsi --safe in the docs directory specified for the project.

We use version 0.12.0 for Jekyll and 2.4.1 for Liquid. It also supports Pygments and RedCloth.

Javadoc

Java docs are built by running find . -name *.java > file.lst && javadoc -d html @file.lst.

If you are unable to fix your build error, please feel free to email us at support@hasdocs.com for assistance.
